🎯 Standards Compliance Testing

These test vectors validate CertNode implementation compliance with established cryptographic standards. Use these test cases to ensure your CertNode integration properly handles both valid receipts and common error conditions.

Usage: Each test vector contains the input files (receipt.json, payload.json, jwks.json) and expected behavior (expected.json). Cross-implementation compatibility is the goal.

🚀 Implementation Guidelines

For Standards Compliance: Your CertNode implementation should pass the "Valid Receipt" test and fail all error condition tests with appropriate error messages as specified in each expected.json file.

Cross-Platform Testing: These test vectors work across all CertNode SDKs (Python, Go, Rust, JavaScript) and can be used to validate any RFC-compliant implementation.

Contribute: Missing a test case? Submit an issue or contribute additional test vectors to improve standards compliance across the ecosystem.
