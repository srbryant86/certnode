Cryptographic Specifications

Digital Signature Algorithm

CertNode uses ECDSA with the P-256 curve and SHA-256 hash function, providing 128-bit security strength as recommended by NIST.

Threat Model & Security Analysis

  • Signature Forgery: Protected by ECDSA cryptographic security
  • Key Compromise: Mitigated by regular key rotation and secure key storage
  • Replay Attacks: Prevented by timestamp validation and unique transaction IDs
