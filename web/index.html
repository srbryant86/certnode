Integration Event Ledger keeps every webhook honest.

Each adapter validates provider signatures, computes idempotent keys, and calls the graph service with parent instructions.

Ledger flow

  • Verify HMAC signature using your CertNode API key
  • Compute `(provider, external_id, checksum)`
  • Persist event and dedupe retries automatically
  • Write receipts via `createReceiptWithGraph`
  • Update parent relationships and fire webhooks

Receipts stay linked even when providers retry

Replays do not create duplicates. Ledger entries surface idempotency statistics, failure reasons, and processing latency so you can monitor ingestion health.
