Open Standard for Tamper-Evident Receipts

CertNode defines a simple, interoperable way to attach cryptographic receipts to any JSON payload. Receipts follow RFC 7515 (JWS) over RFC 8785 (JCS) with ES256, so verification is standard and offline.

Validate a Receipt API Reference SDKs & Examples

Quickstart

  1. npm i @certnode/sdk
  2. POST /v1/sign — get receipt
  3. Verify: verifyReceipt({ receipt, jwks })

Why CertNode

CN

Standards-Based

RFC 7515 JWS and RFC 8785 JCS for interoperable signatures and canonicalization. Works with any compliant implementation.

Offline Verification

Verify using published JWKS — no vendor callback required. Ideal for regulated, air-gapped, or cost-sensitive environments.

Cross-Platform

For Developers

Node.js

Sign and verify receipts programmatically

// install
npm i @certnode/sdk

// sign
const { sign } = require('@certnode/sdk')
const { receipt } = await sign({ payload: { hello: 'world' } })

// verify
const { ok } = await verifyReceipt({ receipt, jwks })
