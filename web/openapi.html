CertNode Technical Specification

Open standard for cryptographically signed digital evidence. RFC-compliant implementation using JWS detached signatures with ECDSA P-256 and JSON Canonicalization Scheme for tamper-evident document integrity.

Protocol Overview

CertNode defines a cryptographic standard for creating tamper-evident digital records. The protocol generates receipts that provide mathematical proof of document integrity and authenticity.

Core Standards

  • RFC 7515: JSON Web Signature (JWS) for cryptographic signatures
  • RFC 8785: JSON Canonicalization Scheme (JCS) for consistent hashing
  • FIPS 186-4: ECDSA with P-256 curve for digital signatures
  • RFC 7517: JSON Web Key (JWK) format for public key distribution

Implementation Requirements

  • Generate JWS detached signatures using ECDSA P-256
  • Apply RFC 8785 canonical JSON serialization before signing
  • Include payload hash (SHA-256) in protected header
  • Support offline verification using published JWKS
