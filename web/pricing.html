Technical Specifications

What cryptographic standards does CertNode use?

CertNode implements RFC 7515 (JSON Web Signature) with ECDSA P-256 digital signatures, RFC 8785 (JSON Canonicalization Scheme) for consistent hashing, and RFC 7517 (JSON Web Key) for public key distribution.

How do I verify a CertNode receipt?

Use any of our SDKs or the WebCrypto API to verify receipts. The process involves validating the JWS signature against the published JWKS using standard ECDSA P-256 verification.

Can I implement CertNode without external dependencies?

Yes! CertNode is an open standard. You can implement it using any cryptographic library that supports ECDSA P-256 and JSON canonicalization. Our SDKs are reference implementations.

Is CertNode compatible with existing systems?

Absolutely. CertNode is designed for integration - use our REST API, SDKs, or implement the standard directly. Works with any programming language and existing infrastructure.

What programming languages are supported?

CertNode SDKs are available for Node.js and browsers. The standard can be implemented in any language with ECDSA P-256 support including Python, Go, Java, C#, and more.

How do I get started with implementation?

Start with our Node.js SDK for quick prototyping, try the browser demo to understand verification, then review the full technical specification for production implementation.
