Technical Security Standards

Cryptographic Implementation

FIPS 186-4 Compliant

ECDSA P-256: Implementation follows NIST FIPS 186-4 standard for digital signature algorithms using the P-256 elliptic curve.

RFC 7515 JWS

JSON Web Signatures: Standards-compliant implementation of detached JWS following IETF RFC 7515 specification.

RFC 8785 JCS

JSON Canonicalization: Deterministic payload normalization using IETF RFC 8785 JSON Canonicalization Scheme.

Key Management

Public Key Rotation: JWKS keys are rotated quarterly with 30-day overlap periods. Historical verification supported for 2 years.

Key Storage: Private keys stored in hardware security modules (HSMs) with FIPS 140-2 Level 3 certification.

Operational Policies

Data Retention

Receipt Storage: Cryptographic receipts and audit logs retained for 7 years to meet regulatory requirements.

Personal Data: Customer personal data retained only as long as necessary for service provision, typically 3 years post-contract termination.

Security Incident Response

Response Time: Security incidents are acknowledged within 4 hours and initial assessment completed within 24 hours.

Customer Notification: Customers notified of incidents affecting their data within 72 hours of discovery.

Service Level Agreements

API Availability: 99.9% uptime SLA for signature and verification endpoints, measured monthly.

Response Time: 95th percentile API response time under 200ms for signature operations.

Privacy & Data Protection

Data Processing

Payload Handling: Customer payloads are processed in-memory only and not permanently stored on CertNode infrastructure.

Audit Logs: Access logs contain only metadata (timestamps, user IDs, operation types) without payload content.

Data Transfers

Encryption in Transit: All data transfers use TLS 1.3 with perfect forward secrecy.

Encryption at Rest: Persistent data encrypted using AES-256-GCM with customer-managed encryption keys where applicable.

Third-Party Assessments

Penetration Testing

Frequency: Annual penetration testing by certified third-party security firms.

Scope: Full infrastructure assessment including API endpoints, key management systems, and supporting infrastructure.

Remediation: Critical and high-severity findings remediated within 30 days of report delivery.

Code Security Review

Static Analysis: Automated security scanning integrated into CI/CD pipeline with mandatory pass requirements.

Dependency Scanning: Third-party dependencies monitored for known vulnerabilities with automated updates for security patches.
